## 1. Introduction
This test plan describes the testing approach for the SauceDemo web application.
The goal is to identify functional, usability, and visual defects.



## 2. Scope of Testing

### In Scope:
- Login functionality
- Product listing and sorting
- Add to cart / Remove from cart
- Checkout process
- Navigation (About page)
- UI and visual elements

### Out of Scope:
- Backend testing
- Security testing
- Performance load testing
- Automation testing


## 3. Test Environment
- Application: https://www.saucedemo.com/
- Browser: Google Chrome
- Operating System: Windows
- Testing Type: Manual Testing


## 4. Test Types
- Functional testing
- UI testing
- Exploratory testing
- Regression testing (basic)


## 5. Test Data
The following test users were used:
- standard_user
- locked_out_user
- problem_user
- performance_glitch_user
- error_user
- visual_user



## 6. Entry Criteria
- Application is accessible
- Test environment is ready
- Test credentials are available



## 7. Exit Criteria
- Critical and high severity bugs are reported
- Core functionalities have been tested
- Test results are documented



## 8. Deliverables
- Bug reports (GitHub repository)
- Test Plan document
- README portfolio file



## 9. Risks and Assumptions
- Demo application contains intentional defects
- Some bugs may be user-specific by design

## 10. Approval
